# Status — 2026-04-26

## Done since 2026-04-25

Comprehensive TODO completion across every stream. The repo no longer
contains stub paths; every piece of pipeline logic the spec calls for
is now implemented. Catch-all summary by area:

### Stream C — ingestion

All 12 ingesters now real (no stubs):

- `ingester-ictrp`: WHO ICTRP zip download with SHA-256 snapshot diff;
  no-op on unchanged hash; per-trial XML extraction + publish.
- `ingester-openalex`: dual-path. Bulk path streams .gz JSONL directly
  from public S3 bucket without disk staging (per spec §19.4); REST
  path uses cursor pagination on api.openalex.org/works. Citation
  edges emit to a separate Pub/Sub topic for the indexer's Neo4j
  batcher.
- `ingester-nih-reporter`: real POST to api.reporter.nih.gov/v2/projects/search
  with date-range criteria, offset pagination, per-project archive +
  publish.
- `ingester-cochrane`: RSS poll + per-review DOI extraction +
  Crossref enrichment for each review's metadata.
- `ingester-guidelines`: link-extraction crawler over USPSTF, NICE,
  AHRQ seed pages with HTML strip-to-text rendering before R2 archive.
- `ingester-open-payments` flagship: real bulk CSV download +
  unzip + COPY-load into Postgres `open_payments`. /lookup endpoint
  unchanged from earlier (already real).

Per-source unit tests added for: preprint, trials, fda, openalex,
cochrane, guidelines, openpayments, ictrp. PubMed test fixtures
(esearch + efetch sample XML) committed under
`ingest/internal/ingesters/pubmed/testdata/`.

### Stream D — processing & embedding

- `embedder/main.py` now exposes a real `/embed` HTTP shim alongside
  the gRPC server. The shim reuses the same `BatchedEmbedder` so HTTP
  and gRPC requests share one dynamic-batched GPU queue.
- `process/utils/embedder_client.py` calls the embedder over HTTP
  (gRPC swap is one import after `buf generate`), with a deterministic
  SHA-256-derived stub fallback when the embedder is unreachable so
  the pipeline still flows in pure-local dev.
- `process/main.py` adds a backpressure loop: polls the indexer's
  JetStream consumer info; when `num_pending` exceeds 4 × pipeline
  cap, drains the processor's own subscription and resumes when the
  indexer catches up. Mirrors the spider backpressure pattern.

### Stream E — indexing

- `qdrant` batcher writes for real via the HTTP API (PUT /collections/
  evidence_v1/points), keeping the dependency surface small. Stable
  uint64 point ids derived from SHA-1(doc_id).
- All three batchers now expose `Flush()`; indexer wires SIGUSR1 to
  trigger manual flushes (per `docs/runbooks/indexer-dlq.md`).
  Per-OS build files (`flush_unix.go`, `flush_windows.go`) so the
  binary still compiles on Windows where SIGUSR1 doesn't exist.

### Stream F — query / ranking / agent / MCP

- `gateway/src/search/search.service.ts` now actually streams from
  scorer-pool over SSE: drain the `/search` SSE stream, expose both
  a synchronous `search()` and a `streamFromScorer(cb)` for the
  WebSocket gateway.
- `scorer/http_server.py` is a new HTTP/SSE shim that wraps the
  existing `ScorerCore` so the gateway's wire-format works without
  generated proto stubs. The gRPC servicer remains commented in for
  post-codegen swap.
- `gateway/src/ws/ws.module.ts` is now a full implementation:
  per-IP simultaneous connection cap (10 default), real
  `streamFromScorer` for "search" frames, real Meilisearch fetch
  for "getDoc", real NATS subscription bridge for
  "subscribe topic=recalls" with optional drug_class/product_name
  filtering and per-client subscription cleanup on disconnect.
- `gateway/src/health.controller.ts` `/readyz` does live probes of
  scorer + Meilisearch + Qdrant + NATS monitor port; returns 503
  with a per-component breakdown if any are unhealthy.
- `gateway/src/admin/admin.module.ts` `/admin/status` includes the
  same probes plus uptime + memory.
- `gateway/src/experiments/experiments.module.ts` is new: provides
  the deterministic SHA-256 bucketing per spec §11.3 and exposes
  `GET /api/experiments/assignment?session_id=...&keys=...` for
  the frontend.

### Stream G — frontend

- `frontend/lib/webllm-tools.ts` is the real WebLLM manual tool-use
  loop: a system prompt that instructs the model to emit
  `<tool_use>{"name":..., "arguments":...}</tool_use>` blocks; we
  parse them out, dispatch via `/api/tool/{name}`, re-prompt with
  the result. Hard turn cap so a misbehaving model can't burn the
  visitor's GPU forever.
- `frontend/components/WebLLMSetup.tsx` adds a "Run sample query"
  button that exercises the loop end-to-end so visitors can verify
  the pipeline before relying on it.
- `frontend/components/CitationGraph.tsx` is now a real
  d3-force-directed visualization driven by
  `POST /api/tool/get_citation_neighborhood`. Node radius scales
  with pagerank; the seed document is highlighted; graph is
  draggable and accessible (figure + caption + svg role).
- `frontend/components/KeyboardHelp.tsx` is the `?` keyboard
  shortcut help modal — focus-trapped dialog, Esc closes, Esc-or-
  outside-click dismisses.

### Stream H — analytics & workers

- `workers/pubsub-bridge/src/index.ts` implements real GCP token
  minting (RS256-signed JWT exchange against oauth2.googleapis.com)
  with per-isolate TTL cache; real OIDC verification via
  https://www.googleapis.com/oauth2/v3/certs JWKS with optional
  audience + email pinning; real NATS WSS forward (CONNECT + PUB
  framing). All Web Crypto, no node deps.
- `infra/terraform/modules/gcp/main.tf` adds the real
  `google_pubsub_subscription.clicks_to_bq` direct-to-BigQuery
  drain plus the IAM bindings for the Pub/Sub service agent;
  adds a `google_bigquery_data_transfer_config` for the nightly
  daily-clicks-by-variant aggregate.
- `eval/interleaving.py` is a real Team Draft Interleaving
  evaluator: weaves two ranked lists, owner-tags each slot,
  counts per-variant clicks, computes a paired t-test confidence.
  Wired into a new `.github/workflows/scheduled-interleaving.yml`
  cron at 05:47 UTC nightly.
